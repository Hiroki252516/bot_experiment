# 01 PRD — 研究用学習支援チャットボット

> 2026-05 update: 教材参照は runtime RAG から **Document Agent Skills** に置換する。ユーザー別 `Skill` は Preference Skill、教材由来の `Document Skill` は ingestion 時に抽出される再利用可能な教材知識として扱う。

## 1. 背景
本研究は、学習者の回答選択とフィードバックを蓄積して、学習者ごとに最適化された回答を返す学習支援チャットボットを対象とする。実装上は、質問→Document Skill / Preference Skill 参照→複数回答候補生成→ユーザー選択→Preference Skill 更新という流れを採用する。

## 2. 研究目的
LLM を用いた学習支援チャットボットにおいて、
- 学習ログ
- ユーザーの回答選択
- ユーザー評価
- ユーザー別 skill 蓄積

がどれだけ個人最適化に寄与するかを検証する。

## 3. 仮説
### 仮説1
Skill によるフィードバック情報の蓄積により、ユーザーが選ぶ回答を予測可能になる。

### 仮説2
Skill による最適化により、主観評価（わかりやすさ、満足度）が向上する。

## 4. MVP の対象範囲
MVP では、以下を「実験可能な最低限機能」と定義する。
1. ユーザーが登録・ログインして Chat を利用できる
2. 学習資料を ingest できる
3. 質問に対して 3 件の候補回答を生成できる
4. ユーザーが 1 件を選べる
5. 満足度・わかりやすさ・自由記述を保存できる
6. 選択結果をもとに skill を更新できる
7. 次回生成時に skill を参照できる
8. 実験条件として skills ON/OFF を切り替えできる

## 5. ユーザーストーリー
### 学習者
- 学習者として、質問を入力したい
- 学習者として、複数候補から最も良い回答を選びたい
- 学習者として、選んだ理由や満足度を送信したい
- 学習者として、自分の過去の会話を見たい

### 研究者
- 研究者として、教材を Document Skill として投入・確認したい
- 研究者として、同一条件下で skills ON/OFF を比較したい
- 研究者として、会話ログと評価データをエクスポートしたい
- 研究者として、skill の更新履歴を確認したい

## 6. 非機能要件
- ローカル Docker 環境で再現できること
- 実験ログが失われないこと
- 参照 Document Skill entry と candidate の関係が追跡できること
- LLM の出力は機械可読な JSON で受け取れること
- 失敗時に再試行またはエラー内容が追跡できること

## 7. スコープ外
- 本格的な認証認可
- 大規模マルチテナント対応
- 音声 UI
- モバイルネイティブアプリ
- 本番運用前提の複雑な分散構成

## 8. 成功条件
### 機能成功
- サンプル教材を入れてエンドツーエンドで動く
- 候補回答生成、選択、保存、skill 更新まで成功する

### 研究成功
- 選択率、主観評価、skills ON/OFF 条件差を分析可能なログが取れる
