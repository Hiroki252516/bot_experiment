# 01 PRD — 適応型学習支援チャットボット（研究用）

本 PRD は、学習ログに基づく適応（Agent Skills）を用いた学習支援チャットボットの研究プロトタイプ要件を定義する。
詳細な研究フロー・ログ仕様は `docs/07_adaptive_learning_design.md` を正本とする。

## 1. 背景
LLM を用いた学習支援において、学習者の行動・テスト結果・質問履歴などの学習ログを蓄積し、それを次の教材提示やテスト生成に反映することで、学習効果が向上する可能性がある。

本プロジェクトは「学習ログに基づく適応型チャットボット」が「非適応型」より学習効果を高めるかを検証するためのローカル実験環境を提供する。

## 2. 研究目的
以下の比較を通じて学習効果（主に Pre/Post の得点差）を評価する。
- Group A: Skills あり（学習ログ駆動で適応）
- Group B: Skills なし（ログは保存するが適応に使わない）
- Group C: 書籍学習（固定テキスト教材、同一フローでテスト実施）

## 3. 仮説（主）
**学習ログに基づく適応型チャットボット（Group A）は、非適応型（Group B）より学習効果を向上させる。**

補助的に、学習過程（ミニテスト推移、教材閲覧時間、テスト所要時間、チャット介入量）が学習効果に与える影響も分析可能な形でログを残す。

## 4. MVP の対象範囲
MVP では、以下を「実験可能な最低限機能」と定義する。
1. ユーザーが `user_id` を自動発行または指定して開始できる（本格認証は不要）
2. 実験 run を開始し、A/B/C の条件を記録できる
3. 学習フローを完走できる（Pre → Cycle1..3 → Post）
   - Pre-test（MCQ）
   - 教材提示（A/Bは生成、Cは固定）
   - 教材閲覧（読了ボタンのみ、時間制限なし）
   - ミニテスト（MCQ、時間制限なし）→採点→理解度推定→次教材
   - 上記を 3 回反復（Cycle 1..3）
   - Post-test（MCQ）
4. 教材閲覧中のみ「教材について質問」できる（A/Bのみ）
5. A では Skills（`skill_profile`）を更新し、次の教材/ミニテスト/チャット回答に反映できる
6. B では Skills を参照/更新しない（= 非適応）
7. 研究用にログをエクスポートできる（CSV/JSONL）

## 5. ユーザーストーリー
### 学習者
- 学習者として、Pre-test を受けて学習を開始したい
- 学習者として、教材を読んで理解を深めたい
- 学習者として、教材閲覧中に不明点を質問したい（A/B）
- 学習者として、ミニテストを解いて理解度を確認したい
- 学習者として、最終テストを受けて学習成果を確認したい

### 研究者
- 研究者として、A/B/C の条件を切り替えて実験 run を実施したい
- 研究者として、Pre/Post の得点差と過程ログ（時間・推移）を比較したい
- 研究者として、ログを CSV/JSONL でエクスポートしたい
- 研究者として、A の skill 更新履歴を確認したい

## 6. 非機能要件
- ローカル Docker 環境で再現できること（`docker compose up --build`）
- 実験ログが失われないこと（DB 永続化）
- LLM の出力は機械可読な JSON で受け取れること
- A/B/C の比較に必要なログ（教材/テスト所要時間含む）が欠けないこと
- 失敗時に再試行またはエラー内容が追跡できること

## 7. スコープ外
- 本格的な認証認可
- 本番運用前提の可用性・スケーラビリティ要件
- RAG/ベクトル検索の優劣比較（本研究の主題ではない）

## 8. 成功条件
### 機能成功
- Pre → Cycle1..3 → Post を 1 user で完走できる
- A/B/C の条件で run を記録できる
- A で skill が更新され、次の生成に反映される
- B で skill が参照/更新されない

### 研究成功
- Pre/Post 差（gain）と、過程ログ（ミニテスト推移、教材閲覧時間、テスト所要時間、チャット介入量）を用いた群比較ができる
